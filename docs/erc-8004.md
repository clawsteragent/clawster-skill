# ERC-8004 On-Chain Identity

## What is ERC-8004?

ERC-8004 is a standard for registering AI agent identities on-chain. Each agent gets a unique numeric ID stored on the BSC (BNB Smart Chain) blockchain, creating a verifiable, immutable identity.

**Contract address**: [`0x8004A169FB4a3325136EB29fA0ceB6D2e539a432`](https://bscscan.com/address/0x8004A169FB4a3325136EB29fA0ceB6D2e539a432)

## Why Agent Identity Matters

- **Verifiability** — anyone can confirm your agent's identity on-chain
- **Accountability** — trades and actions are tied to a registered agent
- **Interoperability** — other protocols and services can reference your agent by ID
- **Uniqueness** — each agent has a globally unique identifier

## Registration Process

During [installation](installation.md), `install.js` handles registration:

1. **Prompts for BSC wallet** — you provide a private key for a wallet with ~0.005 BNB
2. **Calls the ERC-8004 contract** — sends a registration transaction on BSC
3. **Receives Agent ID** — the contract returns a unique numeric ID (e.g., `17763`)
4. **Saves to `.env`** — `AGENT_ID=17763` is written to `~/.openclaw/skills/clawster/.env`

If you already have an Agent ID from a previous registration, you can enter it during install and skip the on-chain transaction.

### Gas Cost

Registration requires ~0.005 BNB for gas (subject to network conditions). This is a one-time cost.

## How Clawster Uses the Agent ID

The Agent ID is stored in the `.env` file and referenced in TOOLS.md config. It serves as the agent's identity when interacting with Aster DEX and other ERC-8004-aware services.
